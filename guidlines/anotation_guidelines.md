Project Overview
The goal of this project is to generate high-quality labeled data to help improve a leading large language model.There are two sequential types of work:
Task Flagging

RLHF Annotations

Both stages operate on the same fundamental unit of work: a task.
What Is a Task?
A task represents a conversation between a user and an AI chatbot. Every task contains the same core components:
System Prompt (optional)
The system prompt is a foundational set of instructions that defines the AI's role, behavior, tone, and constraints for the entire interaction. You can think of it as a “job description” or a pinned ruleset that guides how the AI should respond.
• This may be empty or populated.
• It applies across all turns in the conversation.

Conversation History (optional)
If the interaction includes multiple back-and-forth turns between the user and the AI, those prior turns appear here.
• Single-turn tasks will have no conversation history.
• Multi-turn tasks will include earlier user and assistant messages.

Final User Prompt
This is the final (and sometimes only) question the user asks the AI.

Two Model Responses
Two different responses to the final user prompt are generated by running the prompt through two separate models, producing Response 1 and Response 2, which are then evaluated as part of the task.

These are the outputs that will later be reviewed and evaluated. Both are always present.

Phase 2A: Task Flagging
The first step in Phase 2 is Task Flagging. In this stage, you review each task and categorize it using a set of predefined flags. The purpose of task flagging is to route tasks into the correct buckets so they can later be evaluated by the appropriate persons during RLHF annotation.
Examples of task flags include:
• “Task requires expert legal knowledge.”
• “Task requires expert STEM knowledge.”
• “Task requires recent knowledge (information after February 2025).”
Multiple flags may apply to a single task at the same time. Task flagging does not involve evaluating the quality of the model’s answers—it is strictly about understanding what kind of expertise the task requires.
Detailed instructions for selecting the correct task flags are provided in the Task Flagging section of this guide.
Phase 2B: RLHF Annotations
Once tasks are flagged, they are automatically routed into folders associated with their categories. At that point, the second stage begins: RLHF Annotations. In this stage, you will focus on evaluating the two model responses to the user’s prompt. Your role is to assess how well each response answers the user’s question. Specifically, you will:
• Evaluate factual correctness (is the information accurate and reliable?)
• Identify strengths of each response (clarity, reasoning, completeness, tone, etc.)
• Identify areas for improvement that reduce overall quality
• Compare the two responses and select your preferred response
This process helps train models to better align with human judgment and expert reasoning. Detailed, step-by-step instructions for this stage are provided in the RLHF Annotation section.
Summary: How a Task Moves Through the System
A task is created (user prompt + two model responses).

The task is reviewed during Task Flagging to determine what expertise it requires.

The task is routed to the appropriate folder.

Domain experts perform RLHF Annotations to evaluate and compare the model responses.

This structured workflow ensures that each task is reviewed by the right persons and contributes meaningfully to improving model performance.
