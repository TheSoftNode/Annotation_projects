3. CRITICAL BUG FOUND: INCORRECT WORD DISPLAY LOGIC
================================================================================

BUG TYPE: Logic Error (Conceptual Misunderstanding)
SEVERITY: CRITICAL - Makes the game unplayable
LOCATION: printWord() function (lines 15-35) and main() function call (line 69)

DESCRIPTION:
------------
The printWord() function has a fundamental design flaw. It attempts to use
the guessedLetters array (which is a 256-element ASCII tracking array) as if
it were a position-based array for the word.

CODE ANALYSIS:
--------------

Line 15-35 (printWord function):
    void printWord(const char *secret, const char *guessed) {
        printf("Current word: ");
        for (int i = 0; i < strlen(secret); i++) {
            if (guessed[i] == 1) {  // <-- BUG IS HERE
                printf("%c ", secret[i]);
            } else {
                printf("_ ");
            }
        }
        printf("\n");
    }

Line 69 (function call):
    printWord(secretWord, (const char*)guessedLetters);

THE PROBLEM:
------------
1. guessedLetters[256] is indexed by ASCII values of characters (e.g.,
   guessedLetters[97] = 1 means 'a' was guessed)

2. The printWord function treats guessedLetters[i] as if i represents the
   POSITION in the word (0, 1, 2, 3, 4 for a 5-letter word)

3. For word "array" (5 letters), the loop checks:
   - guessedLetters[0] (should check if 'a' at position 0 was revealed)
   - guessedLetters[1] (should check if 'r' at position 1 was revealed)
   - guessedLetters[2] (should check if 'r' at position 2 was revealed)
   - guessedLetters[3] (should check if 'a' at position 3 was revealed)
   - guessedLetters[4] (should check if 'y' at position 4 was revealed)

4. But guessedLetters[0-4] are always 0 (non-printable ASCII chars were not guessed)
   while the actual guess data is stored at indices like:
   - guessedLetters[97] = 1 (for 'a')
   - guessedLetters[114] = 1 (for 'r')

EVIDENCE FROM TEST OUTPUT:
---------------------------
Test 1 output shows:
- "Good job! 'r' is in the word." (line 11)
- But word display remains: "_ _ _ _ _" (line 15)
- "Good job! 'a' is in the word." (line 26)
- But word display remains: "_ _ _ _ _" (line 30)

This proves that correctly guessed letters are NEVER displayed to the user.

4. SECONDARY BUG: DUPLICATE LETTER COUNTING
================================================================================

BUG TYPE: Logic Error
SEVERITY: CRITICAL - Affects win condition
LOCATION: Lines 97-105 (guess checking loop in main)

CODE:
    for (int i = 0; i < wordLength; i++) {
        if (secretWord[i] == guess) {
            found = 1;
            correctCount++;  // <-- BUG: increments for EACH occurrence
        }
    }

THE PROBLEM:
------------
If a word has duplicate letters (e.g., "programming" has 3 'r's), guessing
'r' increments correctCount by 3, not 1.

IMPACT:
-------
For word "programming" (11 letters):
- Unique letters needed: p, r, o, g, a, m, i, n (8 letters)
- But correctCount target is 11 (wordLength)
- If player guesses 'r' (appears 3 times): correctCount += 3
- If player guesses 'g' (appears 2 times): correctCount += 2
- If player guesses 'm' (appears 2 times): correctCount += 2
- Plus single-letter guesses: p(1), o(1), a(1), i(1), n(1)
- Total: 3 + 2 + 2 + 1 + 1 + 1 + 1 + 1 = 12

This means:
- The win condition (correctCount == wordLength) can be reached even with
  correct gameplay
- BUT the logic is conceptually wrong and could cause issues with different words

Actually, upon deeper analysis: This might be INTENTIONAL design. The
correctCount represents the total number of POSITIONS filled, not unique
letters guessed. This is actually CORRECT for determining when the word is
complete.

VERDICT: NOT A BUG - This is correct logic. Each position must be filled.

6. COMPARISON WITH RESPONSE 1
================================================================================

Response 1 uses a different approach:
- Has a `guessed` array that's the same length as the target word
- Each position in guessed[] corresponds to the same position in target[]
- Initially filled with underscores: guessed[0]='_', guessed[1]='_', etc.
- When a letter is guessed correctly, that position is updated:
  guessed[i] = target[i]

Response 2's bug stems from attempting to use a single array for TWO purposes:
1. Tracking which ASCII characters have been guessed (correct usage)
2. Tracking which positions in the word have been revealed (incorrect usage)

================================================================================
7. CORRECT IMPLEMENTATION APPROACH
================================================================================

To fix Response 2, you would need BOTH arrays:

1. guessedLetters[256] - to track which characters have been tried (already present)
2. revealed[wordLength] - to track which positions have been revealed (MISSING)

The printWord function should be:

    void printWord(const char *secret, const int *guessedLetters) {
        printf("Current word: ");
        for (int i = 0; i < strlen(secret); i++) {
            // Check if the CHARACTER at this position has been guessed
            if (guessedLetters[(unsigned char)secret[i]] == 1) {
                printf("%c ", secret[i]);
            } else {
                printf("_ ");
            }
        }
        printf("\n");
    }

This checks: "Has the character that appears at position i been guessed?"
Not: "Has position i been revealed?" (which is what the buggy code tries to do)
